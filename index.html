The problem(s)

  • Repeat efforts to post to multiple channels

Planning Out the Solution

  • Deciding to go custom and abstract
  • Publication -> Newsletters -> Stories -> Media

Product Demo!

High Level Feature Set

  • Multi-user real time editing
  • Real time previews across channels
  • Simple robust wysiwyg
  • Image cropping
  • Bulk image upload
  • Pretty paste from Microsoft Word
  • CDN integration for fast media retrieveal
  • Configurable image crop properties per publication
  • Custom templates per channel per publication
  • EJS templating per channel engine gives full access to all newsletter data
  • Sortable stories per channel
  • Drag and drop to sort anything!
  • Single Page Application (SPA)
  • Responsive

Why we should consider socket based CRUD

  • Changes the way we design interfaces
  • Practical for a variety of implementations
    • Real time dashboards
    • Collaborative documents
    • Portion of a game experience
